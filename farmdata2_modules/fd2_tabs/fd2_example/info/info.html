FarmData2 Example Module

A sample module for learning about and experimenting with FarmData2 modules.

This module provides examples of how to create and use common UI elements, make API requests and also how to perform Cypress testing. It illustrates a number of the common idioms that are used in FarmData2 modules. It also serves as a template for other modules such as the FieldKit (fd2_field_kit) and the BarnKit (fd2_barn_kit).

Each of the provided sub-tabs is described briefly here:

  • Info:: This page ;)
  • Vars: Illustrates how to access some variables predefined by FarmData2 (e.g. the name and id of the logged in user).
  • Maps: Gives an example of using one of the maps defined for converting between textual names for things and their id's in farmOS (e.g. usernames and user ids, or crop names and crop ids.)
  • API: Demonstrates how to use FarmData2's convenience functions for accessing FarmData2 data through the farmOS API.
  • Cache: SHows how to cache the results of API calls between page visits to improve performance.
  • UI: Provides examples of how to use the specialized UI components that commonly appear in FarmData2 pages.

Each of the above tabs is defined in its own html file (e.g. info.html, vars.html...). A Cypress test suite for each page is included in an associated spec file (e.g. info.spec.html, vars.spec.html).
